# Fork Changelog (customizations)

1. **Group-scoped transfer matching** ‚Äî accounts only auto-match within configured groups; the greedy matcher picks the closest-date pair across all accounts in the same group
2. **Per-family tuning knobs** ‚Äî configurable date window and FX tolerance (defaults match upstream)
3. **Settings UI** ‚Äî `/settings/transfer_match_groups` to manage groups and tuning
4. **Transfer detail panel** ‚Äî shows each transaction's name and a "View transaction" link for both sides of a transfer
5. **Transfer match filter** ‚Äî transactions page filter to show only auto-matched (pending) or confirmed transfer transactions
6. **Self-hosted build pipeline** ‚Äî arm64 image to GHCR, auto-deploys via `sure-deploy`
7. ~~**Rule enrichment ignores locks**~~ ‚Äî **reverted 2026-06-09**, now matches upstream: rules respect attribute locks. A manually-edited (locked) field ‚Äî e.g. a category set from the UI ‚Äî is no longer overwritten by rules on any path (sync, scheduled, or "apply all"). `enrich_attributes`' instance guard skips locked attributes regardless of `source` again, and `family/syncer.rb` applies rules without `ignore_attribute_locks`. The `ignore_attribute_locks` flag now only removes the executor's scope pre-filter; the instance guard still protects locked values.
8. **API: `kind` filter and update** ‚Äî `GET /api/v1/transactions?kind=‚Ä¶` and `PATCH`/`POST /api/v1/transactions` with `kind` now supported; used by the provider-sync scripts to set native kinds at creation (`Contribution ‚Üí investment_contribution`, `Withdrawal ‚Üí funds_movement`). We follow stock Sure P&L semantics ‚Äî there is no `kind`-reclassification housekeeping.
9. **Batch account sync after rule transfer creation** ‚Äî `SetAsTransferOrPayment` now collects affected accounts and calls `sync_later` once per unique account after all transfers are built, instead of once per transaction, preventing N√ó2 sync explosions when rules apply to many transactions
10. **AI auto-categorization uses only subcategories** ‚Äî `Family::AutoCategorizer` filters to subcategories only (`select(&:subcategory?)`), preventing assignment to parent/group categories
11. **Rule action: "Mark as transfer (no counterpart)"** ‚Äî new `SetAsFundsMovement` action executor sets `kind: funds_movement` on matched transactions without creating a synthetic counterpart entry. Useful for single-leg transfers to untracked external accounts (off-ramps, external wallets).
12. **"Investments Interest Returns" dashboard widget** ‚Äî annualized interest yield (ARR) for interest-bearing investment accounts, per-account and portfolio total. An account qualifies if it has any `Interest` investment-activity transaction in the selected dashboard period. ARR = `(period interest / average end_balance) √ó (365 / period_days)`, capital-weighted for the total; annualization is suppressed for periods under 30 days (interest still shown). Fully modular and upstream-merge-safe: new `InvestmentInterestReturnsStatement` PORO + new `pages/dashboard/_investments_interest_returns` partial + dedicated locale file `config/locales/views/pages/investments_interest_returns/en.yml`; the only edits to upstream files are two small additions in `pages_controller.rb` (ivar + one `build_dashboard_sections` entry). Existing widgets are untouched, and the section auto-appends via the existing future-proofing loop.
14. **"Investments Full" dashboard widget** ‚Äî unrealized gains/losses for crypto holdings. Groups all current holdings across visible Investment+Crypto accounts by security, showing per-asset cost basis (converted to family currency at avg cost), current market value, unrealized P&L (amount + %), and portfolio total P&L. Rows sorted by current value. Handles missing cost basis gracefully (shows "-" and a note). Only shown when at least one holding exists. Modular/merge-safe: new `InvestmentsFullStatement` PORO + new `pages/dashboard/_investments_full` partial + dedicated locale file `config/locales/views/pages/investments_full/en.yml`; upstream touch is two small additions in `pages_controller.rb` (ivar + one `build_dashboard_sections` entry).
13. **"Currency breakdown" dashboard widget** ‚Äî net worth distribution by account currency. Groups all visible accounts (included-in-finances for the current user) by their native currency, showing per-currency account count, native net total (assets minus liabilities), the same total converted to the family base currency at today's rates, and each currency's share of net worth (abs-weighted). Vehicle accounts are excluded. Only shown when more than one currency is present. Same modular/merge-safe shape as item 12: new `CurrencyBreakdownStatement` PORO + new `pages/dashboard/_currency_breakdown` partial + dedicated locale file `config/locales/views/pages/currency_breakdown/en.yml`; upstream touch is two small additions in `pages_controller.rb` (ivar + one `build_dashboard_sections` entry).
